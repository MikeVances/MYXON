# SECURITY & SECRETS POLICY (MVP)

## Правила
1. Не хранить секреты в git.
2. Все секреты только через vault/secret manager.
3. Ротация при любом подозрении утечки.
4. Никаких shared root-аккаунтов для production.

## Минимум для MVP
- TLS везде (public ingress only 443).
- Пароли только в хэшированном виде.
- Агентские ключи/токены имеют срок жизни и revocation.
- Аудит логирует: кто, когда, к какому устройству, какой ресурс.

## Инцидент-процедура
1. Отозвать ключ/токен.
2. Выпустить новый секрет.
3. Принудительно закрыть активные сессии.
4. Зафиксировать инцидент в audit и postmortem.
