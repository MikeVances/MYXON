# MVP ACCEPTANCE CHECKLIST

> Обновлено: 04 April 2026 — приведено в соответствие с MYXON_PLATFORM_ARCHITECTURE.md v2
> Связан с: `MYXON_IMPLEMENTATION_PLAN.md` §6

---

## Connectivity (Агент и тоннель)
- [ ] Orange Pi за NAT стабильно подключается к серверу без ручного указания IP контроллера.
- [ ] Авто-дискавери: агент находит контроллер в LAN сам (сканирование порта 5843).
- [ ] При потере связи статус устройства становится offline ≤ 60 сек.
- [ ] Весь внешний доступ работает через 443 (WSS/HTTPS).
- [ ] frpc-тоннель — единственный путь к контроллеру (не прямой IP).

## Device Onboarding (Жизненный цикл устройства)
- [ ] Дилер регистрирует серийник устройства через dealer-портал.
- [ ] Устройство появляется в портале по серийнику со статусом online/offline.
- [ ] **Клиент самостоятельно** активирует устройство через ClaimWizard (серийник с наклейки).
- [ ] Привязка устройства к площадке (Site) клиента работает корректно.
- [ ] Ошибки claim прозрачны: неверный серийник, уже активировано, не найдено.

## Multi-Tenancy (Изоляция тенантов)
- [ ] Тенант A не видит устройства, площадки и аларми тенанта B.
- [ ] Пользователь внутри customer видит только назначенные ему площадки (site-scoped access).
- [ ] **Дилер НЕ видит:** аларм-детали, HMI/экран контроллера, производственные данные, аудит-лог клиента.
- [ ] **Дилер видит:** online/offline статус, количество устройств (для биллинга), имя/контакт клиента.
- [ ] Роли customer_admin / customer_engineer / customer_viewer разграничены корректно.

## Access (Доступ к HMI)
- [ ] Экран контроллера (HOTRACO Remote+) открывается в Chrome/Safari без установки спецПО.
- [ ] WS-бридж корректно проксирует TCP через frpc-тоннель к контроллеру.
- [ ] Сессия имеет TTL и корректно закрывается.

## Alarms (Аларм-система)
- [ ] Аларми контроллера отображаются в AlarmPanel на DeviceDashboard.
- [ ] Фильтрация алармов по severity и state работает.
- [ ] Подтверждение аларма (acknowledge) работает и логируется в аудит.

## Security
- [ ] Нет секретов в репозитории/логах (согласно SECURITY_SECRETS_POLICY.md).
- [ ] Неверный токен агента отклоняется сервером.
- [ ] Все действия доступа пишутся в audit log.
- [ ] Dealer не может получить токен доступа к HMI клиента.

## Ops
- [ ] Dev-окружение поднимается одной командой: `./dev.sh`.
- [ ] Метрики и логи доступны в одном дашборде.
- [ ] Backup/restore БД проверен.
- [ ] Release rollback протестирован.
